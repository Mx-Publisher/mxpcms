This is the mxBB Portal release and installation notes. Further portal information (i.e. roadmap, development news, version history, etc,) is located here.

Overview
mxBB Portal is the only fully modular portal and CMS for phpBB. It features dynamic pages, blocks, and themes, by means of a powerful yet flexible AdminCP. It works without touching phpBB by using integrated features and functions. This makes it a lot easier to upgrade to phpBB 3.0 "Olympus" when the time comes, among other things. mxBB Portal requires phpBB to run, and currently supports the mySQL and postgreSQL databases, with planned support for other database management systems (DBMSs) such as MS-SQL. mxBB Portal is the classic phpBB portal addon, improved and enhanced for every phpBB version released since 2001.

See the demo here running at our site here!
Note that we are running a development version of the next-generation 2.8.0 Core. If you wish to participate in our 2.8.0B1 Quality Assurance forum, take a look at our announcement regarding it.

0. mxBB Portal features

- mxBB is an easy to use and install portal and content management system (CMS)
- Integrates smoothly with the popular phpBB Bulletin Board System (BBS) without modifying (aka MODding) any phpBB files, hence "modular"
- Intuitive, yet advanced Portal Admin Control Panel (AdminCP, or ACP), managing every aspect of the portal, along with the phpBB admin pages
- Modular based approach with easy to use and install/upgrade (and create) add-on packages; this means no MODding to the mxBB Portal core to add features
- Install only the modules with features that you want
- Unlimited number of dynamic portal pages, making unique page designs truly creative
- Flexible Core page blocks featuring standard phpBB themes, BBCode, HTML, languages, and view/edit permissions (user/group)
- Additional Module page blocks, enriched with tailored features unique to each module
- Advanced interface for categorized menu navigation
- Cache system for fast functionality and decreased DB queries needed
- Support forum (www.mx-system.com)

Included Modules in the mxBB Portal Core distribution
- Last Message Module
- Statistics Module
- TextBlocks Module

Requirements
- PHP4 (Recommended: latest version)
- phpBB 2.0.x (latest version)
- mySQL or postgreSQL database
- An FTP client capable of uploading files to your server (this is not needed if you're hosting your own server)

Note that the mxBB Portal Core 2.7.x and up do NOT support PHP5. Sorry. We are planning to implement PHP5 support with our mxBB 3.0 release, which will be released parallel to phpBB 3.0 "Olympus".

Be sure to use the latest phpBB, mxBB Core, and module releases; otherwise, conflicts may occur!

Ia. Installation Instructions

1) mxBB Portal requires a working installation of phpBB and will not run standalone. If phpBB is not installed on your system, please visit www.phpbb.com for further instructions. For the new installer introduced in 2.7.5, you need not set the board's default theme to subSilver; you also need not set the default language to English. The installer automatically picks all of this information for you.

2) The mxBB Portal and phpBB forum must be installed in different directories. Do not mix the files up!!

Note that for ALL of the directory structure examples below, you MUST include the trailing slash at the end of the folder names! If you don't, you'll run into errors later!


Default directory structure is:

/your/path/to/public/html/YOUR_FORUM_NAME/ (this is the place where your working phpBB installation should be located)

/your/path/to/public/html/YOUR_PORTAL_NAME/ (this is the place were all mxBB files should be located)

Note that you can alternatively install mxBB in your root directory (which can be accessed via www.yourdomain.com/) while your forum can be installed in a folder within mxBB's (ie it'd be located in www.yourdomain.com/phpBB2/).

Using the default setup above, the install path variables should be:

phpBB Relative Path: ../YOUR_FORUM_NAME/
Portal URL: http://www.yourdomain.com/YOUR_PORTAL_NAME/
phpBB URL: http://www.yourdomain.com/YOUR_FORUM_ NAME/

Note #1: If you move the phpBB folder into another one before installing mxBB, the phpBB configuration (the /phpbbroot/config.php file) must also be modified to match the new phpB location. This is done in the Admin CP and the config.php file directly.

Note #2: phpBB and mxBB use separate config.php files. Do not confuse them and mix them together! They are two different files.

Example: mxBB is installed in your root directory

/your/site/root/

/admin
/db
/language
/modules
/phpBB2 (this is the place where your working phpBB installation should be located)
/templates
/*.*

Note: These are standard install paths, since they acknowledge the standard phpBB install directory (phpBB2/). No further changes are needed for the phpBB configuration.

3) Unzip the distributed mxBB Portal files and upload

Special note
mxBB needs write access/permissions to the cache folder and the mx_meta.inc file.

## CHMOD (0777) THE FOLLOWING FILES/DIRECTORIES:
## - mxroot/cache/
## - mxroot/mx_meta.inc

For those with Windows servers, CHMOD 0777 gives read, write, and access permissions to all 3 auth types (commonly Owner, Group, and Other/Anonymous). You'll have to give the files/folders above these permissions; this depends from host to host, though usually you can do it via your hosting control panel (ex. cPanel, DirectAdmin, etc.).

4) In your browser, call http://www.yourdomain.com/mxbbroot/install/mx_install.php. Replace "mxbbroot" with whatever name you had for your mxBB folder, which contains all of the mxBB files and folders.

With the new installer recently introduced for 2.7.5, all of the values should already be filled in, and you should be set up to go and click "Install." However, you should go over the information filled in. If you definitely need something changed, then go ahead and see this document for details on how to set the installer to allow you to enter your own settings. Note that you should only do this if you know what you are doing.

The aforementioned document describes this: "To choose your own configuration, as opposed to the installer's suggestions, open up the mx_install.php file in your favorite PHP or text editor:

Find:

Code:
define('INSTALL_READONLY', true);

Replace With:

Code:
// define('INSTALL_READONLY', true)

Basically, you're commenting out that line, which will in turn allow you to set up your own configuration. "

Note: The install script creates/generates the mxBB config.php file. If for some reason the install fails, you may need to remove the generated config.php manually before retrying the install. Also, if you feel like doing a fresh reinstallation, you have to remove old mxBB config.php file; otherwise, the script will try to update.

Also, some webservers deny the installer the rights to edit the config.php file. In this case you'll need to CHMOD the config.php file to 0777. (CHMOD is overviewed above). If you have any other errors regarding config.php (you shouldn't), go here.

5) Don't forget to remove the install/ and contrib/ folders after a successful installation! Otherwise, this could mean a potential security hole.

6) Installation Complete!
Now enter the Portal AdminCP and configure the portal. The user interface is intuitive. For further information, visit some of our documents and Knowledge Base articles at mx-system.com.



Note 1: In version 2.7.x the mxBB config.php file has a new format. This may be of interest for those who want to edit this file directly:
Code:
<?php

  //
  // MX-Portal 2.x auto-generated config file
  //
  //

  $mx_table_prefix = "mx_";

  define('MX_INSTALLED', true);
 
  $dbms = 'mysql';

  $dbhost = 'host';
  $dbname = 'DBNAME';
  $dbuser = 'DBUSER';
  $dbpasswd = 'DBPASSWD';

  $table_prefix = 'phpbb_';

  define('PHPBB_INSTALLED', true);

  $phpbb_root_path = $mx_root_path . "/forum/"

?>

Ib. Upgrade Instructions



Note: This screenshot is from the 2.7.6 release; however, this 2.7.7 release only introduced security fixes; as such, nothing changed on the user's end. When upgrading from older versions, you should first apply the changes described to upgrade from your current version to the next, and so on!

When upgrading from older versions, you should first apply the changes described to upgrade from your current version to the next, and so on!

If your current portal version is far from more than one release, we advise you to remove all your portal files from your webspace, then proceed as if it was a new installation, except: keep your current portal config.php file (otherwise, the installation wizard will assume you're doing a fresh installation! ) and also all non-core modules in place.

Remember: The rule that makes the Installation Wizard switch from installation to upgrade modes is the presence of a valid config.php in your portal root folder, with all correct settings in place. The upgrade mode makes changes to your database to upgrade to latest version, whereas the install mode DELETES all your previous portal tables (if exist) and then creates fresh tables with default settings!

Please, make sure you have verified backups of ALL your files and database BEFORE proceeding!

Now, once you have done all of the above, use one of the bolded green steps below.

Upgrading using the Full Package

There are two ways to upgrade using the Full Package. They are as follows:

Method #1.
  1. Remove all of your mxBB files from your webspace, except for config.php and any other files you want to keep, but are not affected by the upgrade.
  2. Download the Core Full Package. UnZIP it.
  3. Upload all of the files in the Full Package into your webspace, except for config.php.
  4. Run http://www.yourdomain.com/mxbbroot/install/mx_install.php to upgrade.
  5. Once the above step has been done, remove the install/ and contrib/ directories.
  6. You're done!

Method #2.
  1. Download the Core Full Package. UnZIP it.
  2. Find the appropriate HTML upgrade document in the install/ folder. The naming convention is as follows:"mx_upgrade_changes_between_2.7.x_and_2.7.x.htm". Please follow all of the instructions located in those upgrade documents. The list of these available documents is as follows:

    • Upgrade from 2.7.3 to 2.7.5
    • Upgrade from 2.7.5 to 2.7.5-pl4
    • Upgrade from 2.7.5-pl4 to 2.7.6
    • Upgrade from 2.7.6 to 2.7.6-pl1
    • Upgrade from 2.7.6-pl1 to 2.7.6-pl2
    • Upgrade from 2.7.6-pl2 to 2.7.7

  3. The previous step covers all of Method #2. Make sure you've done everything those documents instruct you to do.
  4. You're done!

Upgrading using the Changed Files Package
  1. Download the Core Changed Files Package. UnZIP it.
  2. Find the appropriate folder that matches the upgrade path you're attempting. For example, if you wish to upgrade from mxBB 2.7.5 to 2.7.7, you should find the folder named "2.7.5 to 2.7.7".
  3. Upload the contents of the appropriate folder into your root mxBB folder, which is on your webspace. Make sure your FTP client "knows" to overwrite the old files with the new ones.
  4. Finish the upgrade by running http://www.yourdomain.com/mxbbroot/install/mx_install.php .
  5. Remove the install/ directory from your webspace.
  6. You're done!
 Upgrading using the Code Changes Package
  1. Download the Code Changes Package.
  2. Find the appropriate text file that has the upgrade path you're using. The naming convention is as follows: "cc_27x-27x.txt"; the first "27x" represents the version you're currently running, while the second "27x" is the "target version" (ie the one you're going to upgrade to).
  3. Read through the install script's header before proceeding. Pay special attention to the "MOD Description" and "Author's Notes" fields.
  4. Do everything the install script says.
  5. Upload the install/ folder that was distributed in the Code Changes package into your root mxBB folder.
  6. Finish the upgrade by running http://www.yourdomain.com/mxbbroot/install/mx_install.php .
  7. Delete the install/ folder.
  8. You're done!
Note: If you are upgrading from mxBB 2.704, note that we have renamed most of the core images and moved them to the template's folder. This will affect nav images in the menu, and Page Admin. You'll have to update the image names manually!! There is a PhotoShop template file ( *.PSD ) in the mxBB Download area with all mxBB graphics if you want to develop your own styles; it is located here.

IV. Additional Languages

First, check to see if your language is already translated. Translated languages are downloaded in the mxBB download area. The translations/language packs are released and maintained here.

If your language has not already been translated, you can create a translation yourself! Simply duplicate (copy and paste) the /language/lang_english/ directory, rename it to something like "lang_yourlanguage." Translate all of the included files, save it, ZIP it, and upload it to the mxBB Languages Download Area. Be sure to report translated languages to us. We appreciate your contribution.
