Overview
Mx system is a fully modular portal, for phpbb, while it uses integrated features it works without touching phpbb. Mx-system requires phpbb to run, and currently supports the mySQL and postresSQL databases. The system is still in development and as such may be unsuitable for novices.

0. MX Portal features

- Easy to use and install web portal
- Intergrates smoothly with the popular phpbb forum, without modifying any phpbb files
- Intuitive, yet advanced Portal Admin Panel
- Modular based approach with easy to use and install (and create) addon packages
- Portal defined "pages", referred to by pageid; featuring view/edit permissions based on user/group belongings and customizable overall_header file
- Page blocks compatible with phpbb themes, languages and permissions
- Categorized menu navigation defined by user rank

Ia. Installation Instructions


1) The Mx Portal requires a working installation of PHPBB and will not run standalone. If PHPBB is not installed on your system, please visit www.phpbb.com for further instructions.

Also, select the SubSilver theme and English language for your phpbb forum (admin and default) before installing the portal - otherwise the install script may behave incorrectly.

2) The Mx Portal and phpBB forum is installed in different directories. Do not mix the files up!!

Default directory structure is:

MyDomain/[MY_FORUMNAME] (this is the place where your working PHPBB installation should be located)
MyDomain/[MY_PORTALNAME] (this is the place were all mx files should be located)

Using this setup, the install path variables should be:

PHPBB PATH: ../MY_FORUMNAME/
PORTAL URL:http://MyDomain/MY_PORTALNAME/
PHPBB URL:http://MyDomain/MY_FORUMNAME/

Note1: If you move the phpbb installation to another folder, the phpbb configuration (phpbb2/config.php file) must also be modified to match the new phpbb location. This is done either in the Admin control panel and/or in the config.php file directly.

Note2: phpBB and MX uses individual config.php files. Do not confuse them together!

Example 1:

MyDomain-root/(MX files...)
/admin
/DB
/phpbb2 (this is the place where your working PHPBB installation should be located)
/language
/modules
/templates



Using this setup, the install path variables should be:

PHPBB PATH: phpbb2/
PORTAL URL:http://MyDomain/
PHPBB URL:http://MyDomain/phpbb2/



Note: These are standard install paths, since they acknowledge the standard phpbb install directory (/phpbb2). No further changes are needed for the phpbb configuration.

Example 2:

ROOT/

/phpbb2 (this is the place where your working PHPBB installation should be located)
/mx (this is the place where your mx installation should be located)




Using this setup, the install path variables should be:

PHPBB PATH: ../phpbb2/
PORTAL URL:http://MyDomain/mx/
PHPBB URL:http://MyDomain/phpbb2/



Note: These are standard install paths, since they acknowledge the standard phpbb install directory (/phpbb2). No further changes are needed for the phpbb configuration.

3) Unzip the distributed MX-Portal files and upload

Special note
MX needs write access to certain files/folders: mx_meta.inc and cache/

## Create and CHMOD (0777) THE FOLLOWING DIRECTORIES:
## - mxroot/cache/
## - mxroot/mx_meta.inc

4) In your browser, call [PORTAL URL]/install/ and follow instructions.

Note: The install script features a 'validate setup' option. Be sure all entries/settings all cleared (green) before installing! Do not force an installation, unless you search for problems later on...

Note: The install script creates/generates the MX config.php file. If for some reason the install fails, you may need to remove the created config.php manually before retrying install.

Note: If you feel like doing a fresh reinstallation, you have to remove old mx config.php file, otherwise the script will try to update.

Also, some webservers deny the script to create the config.php. In this case you need to first upload an empty config.php file to be udated by the install script. If this also fails the install script will send you the config.php file and you have to upload it manually.

5) Verify the main menu (navigation) configuration. If you have chosen a non-standard directory structure you must reconfigure certain menu options. This is done in the Admin Control Panel.

6) Don't forget to remove the install folder after successful installation! Otherwise this could mean a potential security hole...

7) Installation complete.
Now enter the portal adminCP and configure the portal. The user interface is intuitive. For further information, visit some of our docs/kbs at mx-system.com.



Note 1: In version 2.7 there's a new syntax for the Mx Portal config.php file. This may be of interest for those who want to edit this file directly...

Code:
<?php

  //
  // MX-Portal 2.x auto-generated config file
  //
  //

  $mx_table_prefix = "mx_";

  define('MX_INSTALLED', true);
 
  $dbms = 'mysql';

  $dbhost = 'host';
  $dbname = 'DBNAME';
  $dbuser = 'DBUSER';
  $dbpasswd = 'DBPASSWD';

  $table_prefix = 'phpbb_';

  define('PHPBB_INSTALLED', true);

  $phpbb_root_path = $mx_root_path . "/forum/"

?>

III. How to syncronize the mx and phpBB overall_headers

For flexibility, MX and phpBB use its own overall_header.tpl (and overall_footer.tpl), contained within each template theme's folder. If you want the same layout/feel when browsing between mx and phpbb, you need to edit the phpbb overall_header.

Note: MX use the same overall_header syntax as phpbb, please be sure you understand the phpbb approach before trying to adapt MX using it.

Individual Headers
The Portal and the Forum use individual (their own) headers and this is the default mode after installation. This means when browsing to a forum page the portal look and feel is replaced by the forum layout. Nothing further need to be done...

Editing the phpBB overall_header
To get the mx header also for non-portal pages (i.e forum pages), you need to overwrite the phpbb overall_header.tpl with a modified mx overall_header.tpl. Such an example file is included in the contrib folder.

Then you have to add some mx definitions in the phpBB page_header.php (required) and lang_main.php (optional)

OPEN phpbb/includes/page_header.php

FIND

Code:
   'U_GROUP_CP' => append_sid('groupcp.'.$phpEx),


AFTER ADD

Code:
//
// MX Definitions for syncronized overall_header
//
   // Define the relative path to mx, eg ../mx/ or ../ or whatever
   'MX_ROOT_PATH' => '../core/',
   'TEMPLATE_ROOT_PATH' => 'templates/'.$theme['template_name'],
   'L_FORUM' => $lang['Forum'],
   //
   // Use either 'sitename' or manual definfition in lang_main for the home link text!
   // Note: if using $lang['Portal_home'], you have to add the entry in lang_main youself
   // for example:
   // $lang['Portal_home'] = 'Portal Home';
   // before // That's all, Folks!
   //
//   'L_HOME'  => $board_config['sitename'],
   'L_HOME'  => $lang['Portal_home'],
   'U_INDEX_FORUM'    => append_sid( 'index.'.$phpEx . '?page=2'),
   'U_INDEX' => append_sid( 'index.'.$phpEx),
   // Additional header links are defined accordinly
//
// END MX
//


If using the user defined L_HOME entry, add the lang def to lang_main (optional)

OPEN phpbb/language/.../lang_main.php

FIND

Code:
//
// That's all, Folks!


BEFORE ADD

Code:
// MX addon for syncronized overall_header
$lang['Portal_home'] = 'Portal Home';


For further info on how to modify headers...consult our support forum.

IV. Additional themes/styles

Installed phpBB themes/templates may be used for MX too, but they also need to be installed for MX. Lets say you want to use a theme called 'Mytheme'.

Note: MX use the same theme/style syntax as phpbb, please be sure you understand the phpbb approach before trying to adapt MX using it!

- The theme/style 'Mytheme' (for example 'BlueSilver') must first be installed for phpbb (the forum). Visit www.phpbb.com for further instructions.
- Duplicate (copy and paste) the MxPortal/template/SubSilver directory and rename the directory 'Mytheme'.
- If you like to keep the Mx header, the header images are located in the template/Mytheme/images/ folder. These images may be replaced by your own to match your style.
- If you like to keep the PHPBB header, copy the PHPBB/overall_header.tpl and PHPBB/admin/page_header.tpl files to your own template folder, i.e. MxPortal/template/Mytheme.
Then edit the MyTheme/overall_header.tpl and Mytheme/admin/page_header.tpl files.
Search for "templates/subSilver/" and "../templates/subSilver/". Replace these paths with "{U_PHPBB_ROOT_PATH}{TEMPLATE_ROOT_PATH}"
- Make sure to change the css link in the overall_header.tpl file to
Code:
<link rel="stylesheet" href="{U_PHPBB_ROOT_PATH}{TEMPLATE_ROOT_PATH}{T_HEAD_STYLESHEET}" type="text/css">

- Rename the *.css file to Mytheme.css (optional)
- Test the new template (theme/style).
